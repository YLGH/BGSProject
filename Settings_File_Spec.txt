Each line contains zero or one key-value pair, separated by one or more non-alphanumeric characters.
Comments have # as the first alphanumeric character on the line, and are ignored.
Alphanumeric characters are defined to be 0-9, A-Z, a-z and underscore (_).
Leading non-alphanumeric characters are ignored.
Keys may contain alphanumeric characters only.
Values may contain any character but must start with an alphanumeric character.
Both keys and values may be up to 20 characters long.
Keys and preset values (e.g. file types, Booleans) are not case-sensitive.
